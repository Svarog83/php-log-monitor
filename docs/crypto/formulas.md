# Crypto Tracker - Formula Reference

All formulas use ranges $2:$1000 to avoid full-column references for performance.

## EUR Rate Lookup (shared pattern)

Used in FiatOperations (H), Trades (M), Transfers (H), Portfolio (F).

```
=IFERROR(INDEX(SORT(FILTER(Rates!A:C, Rates!B:B=<asset>, Rates!A:A<=<date>), 1, FALSE), 1, 3), "")
```

Logic:
1. FILTER: Get all Rates rows where asset matches AND date ≤ transaction date
2. SORT by date descending (column 1, FALSE)
3. INDEX row 1, column 3: Take the EUR rate from the most recent matching row

## FiatOperations Formulas

| Column | Name | Formula (row 2 example) |
|--------|------|------------------------|
| G | Price Per Unit | `=IF(F2>0, D2/F2, "")` |
| H | EUR Rate | `=IFERROR(INDEX(SORT(FILTER(Rates!A:C,Rates!B:B=C2,Rates!A:A<=A2),1,FALSE),1,3),"")` |
| I | Fiat Amount EUR | `=IF(AND(D2<>"",H2<>""), D2*H2, "")` |
| L | Fee EUR | `=IF(AND(J2<>"",H2<>""), J2*H2, "")` |

**Summary block** (below data, auto-formatted):
- Total Buy EUR: `=SUMIFS(I:I, B:B, "Buy")`
- Total Sell EUR: `=SUMIFS(I:I, B:B, "Sell")`
- Difference EUR: Buy − Sell

## Trades Formulas

| Column | Name | Formula (row 2 example) |
|--------|------|------------------------|
| D | Base Asset | `=LEFT(C2, FIND("_",C2)-1)` |
| E | Quote Asset | `=MID(C2, FIND("_",C2)+1, LEN(C2))` |
| J | Order Amount | `=H2*I2` |
| M | EUR Rate | `=IFERROR(INDEX(SORT(FILTER(Rates!A:C,Rates!B:B=E2,Rates!A:A<=A2),1,FALSE),1,3),"")` |
| N | Amount EUR | `=IF(AND(J2<>"",M2<>""), J2*M2, "")` |
| O | Fee EUR | `=IF(AND(K2<>"",M2<>""), K2*M2, "")` |

Note: EUR Rate looks up the **quote asset** rate (e.g., USDT for BTC_USDT pair).

## Transfers Formulas

| Column | Name | Formula (row 2 example) |
|--------|------|------------------------|
| H | EUR Rate | `=IFERROR(INDEX(SORT(FILTER(Rates!A:C,Rates!B:B=B2,Rates!A:A<=A2),1,FALSE),1,3),"")` |

## FIFOLots Formulas

| Column | Name | Formula (row 2 example) |
|--------|------|------------------------|
| H | Total Cost EUR | `=F2*G2` |
| J | Days Held | `=IF(B2<>"", TODAY()-B2, "")` |
| K | Tax-Free? | `=IF(J2>365, "Yes", "No")` |

### Lot Source References
- `FiatOp #N` — Buy from FiatOperations row N
- `Trade #N` — Buy from Trades row N (base asset received)
- `Trade #N (quote)` — Sell from Trades row N (quote asset received)

## Portfolio Formulas (auto-generated by Apps Script)

### Balance (Column C) — Aggregation from All Data Sheets

```
= SUMIFS(FiatOps bought at wallet)
- SUMIFS(FiatOps sold at wallet)
+ SUMIFS(Trades bought base asset at exchange)
- SUMIFS(Trades sold base asset at exchange)
+ SUMIFS(Trades sold quote = this asset at exchange)
- SUMIFS(Trades bought quote = this asset at exchange)
+ SUMIFS(Transfers incoming to wallet)
- SUMIFS(Transfers outgoing from wallet)
- SUMIFS(Transfer fees from wallet for this asset)
```

Full formula (row r, A=Asset, B=Wallet):
```
=SUMIFS(FiatOperations!F$2:F$1000, FiatOperations!O$2:O$1000,$Br, FiatOperations!E$2:E$1000,$Ar, FiatOperations!B$2:B$1000,"Buy")
-SUMIFS(FiatOperations!F$2:F$1000, FiatOperations!O$2:O$1000,$Br, FiatOperations!E$2:E$1000,$Ar, FiatOperations!B$2:B$1000,"Sell")
+SUMIFS(Trades!I$2:I$1000, Trades!B$2:B$1000,$Br, Trades!D$2:D$1000,$Ar, Trades!G$2:G$1000,"Buy")
-SUMIFS(Trades!I$2:I$1000, Trades!B$2:B$1000,$Br, Trades!D$2:D$1000,$Ar, Trades!G$2:G$1000,"Sell")
+SUMIFS(Trades!J$2:J$1000, Trades!B$2:B$1000,$Br, Trades!E$2:E$1000,$Ar, Trades!G$2:G$1000,"Sell")
-SUMIFS(Trades!J$2:J$1000, Trades!B$2:B$1000,$Br, Trades!E$2:E$1000,$Ar, Trades!G$2:G$1000,"Buy")
+SUMIFS(Transfers!C$2:C$1000, Transfers!E$2:E$1000,$Br, Transfers!B$2:B$1000,$Ar)
-SUMIFS(Transfers!C$2:C$1000, Transfers!D$2:D$1000,$Br, Transfers!B$2:B$1000,$Ar)
-SUMIFS(Transfers!G$2:G$1000, Transfers!D$2:D$1000,$Br, Transfers!H$2:H$1000,$Ar)
```

**Note:** FiatOperations col O = Destination Wallet, col N = Chain.
Transfers col G = Fee amount, col H = Fee Asset, col F = Chain.

### Avg Cost EUR (Column D) — Global Per Asset

```
=IFERROR(
  SUMIFS(FIFOLots!H$2:H$1000, FIFOLots!C$2:C$1000, $Ar)
  / SUMIFS(FIFOLots!F$2:F$1000, FIFOLots!C$2:C$1000, $Ar),
  "")
```

This is a **global weighted average** across all wallets for the asset:
- Numerator: Sum of (Qty Remaining × Cost Per Unit) for all lots of this asset
- Denominator: Sum of Qty Remaining for all lots of this asset
- Ignores wallet filter to handle lots that stayed at purchase wallet after transfer

### Header row (Row 1) — filter-aware subtotals

Column headers C, E, G, H, I show the label on the first line and the sum (or ratio) of **visible (filtered) rows** on the second line (SUBTOTAL(109, …)). No separate TOTAL row; all totals are in the header. Rounding: Balance 5 decimals, EUR 2 decimals. Unrealized P/L % = sum(H)/sum(E) when sum(E)≠0.

| Column | Header formula |
|--------|----------------|
| C | `="Balance" & CHAR(10) & ROUND(SUBTOTAL(109,C2:C1000), 5)` |
| E | `="Total Cost EUR" & CHAR(10) & ROUND(SUBTOTAL(109,E2:E1000), 2)` |
| G | `="Current Value EUR" & CHAR(10) & ROUND(SUBTOTAL(109,G2:G1000), 2)` |
| H | `="Unrealized P/L EUR" & CHAR(10) & ROUND(SUBTOTAL(109,H2:H1000), 2)` |
| I | `=IF(SUBTOTAL(109,E2:E1000)=0,"Unrealized P/L %" & CHAR(10) & "","Unrealized P/L %" & CHAR(10) & ROUND(SUBTOTAL(109,H2:H1000)/SUBTOTAL(109,E2:E1000), 2))` |

Set automatically by **Refresh Portfolio** / **Refresh All** (row height 48px, wrap on).

### Other Portfolio Columns

| Column | Name | Formula |
|--------|------|---------|
| E | Total Cost EUR | `=C×D` (balance × avg cost) |
| F | Current Rate EUR | EUR rate lookup (latest rate for asset) |
| G | Current Value EUR | `=C×F` (balance × current rate) |
| H | Unrealized P/L EUR | `=G-E` (value − cost) |
| I | Unrealized P/L % | `=IFERROR(H/E, "")` |
| J | Last Purchase Date | `=IFERROR(TEXT(MAX(FILTER(FIFOLots dates, asset=$A)), "YYYY-MM-DD"), "")` |
| K | Avg Holding Days | `=IFERROR(SUMPRODUCT(remaining_qty × days_held) / SUM(remaining_qty), "")` |
| L | Qty >1 Year | `=SUMPRODUCT(remaining_qty × (days>365))` |
| M | Tax-Free Value EUR | `=L×F` |

## ChainBalances Formulas (auto-generated by Apps Script)

### Self-Custody Wallets (chain breakdown)
Uses `buildChainBalanceFormulas_()` — same SUMIFS structure as Portfolio Balance but with additional chain filter on FiatOperations and Transfers.

### Exchanges (total balance, chain="All")
Uses `buildExchangeBalanceFormulas_()` — same SUMIFS structure as Portfolio Balance without chain filter.

### Common columns (both types)
| Column | Formula |
|--------|---------|
| D | Balance (SUMIFS, same structure as Portfolio) |
| E | Current Rate EUR (latest rate lookup) |
| F | Value EUR (`=D×E`) |

## Adding Formulas to New Rows

When the Apps Script is installed, use **Refresh All** — formulas are auto-generated.

For manual additions (if needed), copy formulas from the row above:
- **FiatOperations**: G, H, I, L
- **Trades**: D, E, J, M, N, O
- **Transfers**: H, I
- **FIFOLots**: H, J, K (but prefer using Sync FIFO Lots)
- **Portfolio**: C through M (but prefer using Refresh Portfolio)
